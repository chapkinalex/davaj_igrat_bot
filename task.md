# Задача для Cursor-агента

## Инструкция
Ты — агент Cursor. Твоя задача — внести изменения в код проекта по спецификации ниже.

Шаги:
1. Внимательно прочитай [TASK] и [USER_IDEA].
2. Построй подробный план изменений по файлам и функциям.
3. Внеси изменения в код проекта.
4. В конце выдай список изменённых файлов и как проверить результат.
5. Запиши короткий отчёт в файл task_result.txt в корне проекта (итог, список изменённых файлов, как проверить) — его получит пользователь в Telegram.

## Спецификация

[TASK]
Краткое описание: сделать изменения в проекте «Давай играть» по запросу пользователя.

[USER_IDEA]
давай сформируем приветственное сообщение

[ANALYSIS]
Пользователь описывает изменение или новую функцию для проекта «Давай играть». Требуется превратить это в задачу для разработки.

[CONTEXT]
Проект: «Давай играть» — телеграм-бот (и в будущем другие интерфейсы),
который помогает родителям подбирать игры и фокусы для ребёнка
на основе диагностики поведения и семейной ситуации.

Участники процесса:
- Пользователь (ты) — формулирует идею/задачу.
- Архитектор (Perplexity) — помогает продумать структуру и варианты решения.
- Cursor (AI-кодер) — вносит изменения в код по этому ТЗ.

[REQUIREMENTS]
- Внимательно перечитать блок [USER_IDEA].
- Ясно сформулировать цель изменения (что изменится для родителя/ребёнка).
- Предложить 1–2 варианта реализации (минимальный и улучшенный).
- Ясно указать, какие файлы и модули в проекте нужно изменить или создать.
- Описать, какие данные уже есть и какие нужно добавить (модели, поля, состояния).
- Не менять общую архитектуру проекта без явного указания.

[ACCEPTANCE]
- Реализация удовлетворяет описанным требованиям.
- Сценарий из [USER_IDEA] можно пройти от начала до конца без ошибок.
- Поведение понятно пользователю (нет неожиданных шагов и сложных формулировок).
