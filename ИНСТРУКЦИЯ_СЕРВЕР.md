# Пошаговая инструкция: запуск бота на сервере

Делайте всё по порядку. Не пропускайте шаги.

---

## ЧАСТЬ 1. Подготовка: проект на сервере

Сначала нужно, чтобы папка с ботом оказалась на сервере. Выберите **один** способ.

### Способ А. Через GitHub (если проект уже в репозитории)

1. Убедитесь, что ваш проект запушен в GitHub (все файлы закоммичены и отправлены).
2. На сервере будете вводить команды в **Части 2** — там есть шаг «склонировать репозиторий». Пока просто запомните: понадобится **ссылка на репозиторий** (например `https://github.com/ваш_логин/davai_sigrai_bot`).

### Способ Б. Через Cursor (если без GitHub)

В **Части 2** вы подключитесь к серверу и откроете папку (например `/home/ubuntu`). Потом откроете **второе окно** Cursor с вашей **локальной** папкой бота и перетащите все файлы в папку на сервере. Подробно — в шагах ниже.

---

## ЧАСТЬ 2. Подключение к серверу в Cursor

### Шаг 2.1. Открыть палитру команд

- Нажмите на клавиатуре: **Ctrl + Shift + P** (три клавиши вместе).
- Вверху появится строка ввода.

### Шаг 2.2. Подключиться по SSH

- В этой строке начните печатать: **connect to host**.
- В списке выберите пункт **Remote-SSH: Connect to Host...** (стрелками или мышкой, затем Enter).
- Если спросят «Select SSH Host» — введите хост в формате: **логин@адрес_сервера**.
  - Примеры: `ubuntu@192.168.1.100`, `root@myserver.ru`, `ubuntu@ваш-ip-или-домен`.
  - Логин и адрес вам дал тот, кто настроил сервер (или вы сами создавали).
- Нажмите **Enter**.

### Шаг 2.3. Новое окно и пароль

- Откроется **новое окно** Cursor.
- Может появиться выбор платформы — выберите **Linux**.
- Когда попросит пароль — введите **пароль от сервера** (тот, что для этого логина). При вводе символы не отображаются — это нормально, введите и нажмите **Enter**.
- Подождите, пока в левом нижнем углу окна появится надпись вида **SSH: логин@адрес** — значит, вы на сервере.

### Шаг 2.4. Открыть папку на сервере

- В этом же окне (где уже SSH) нажмите **Ctrl + Shift + P**.
- Введите: **open folder**.
- Выберите **File: Open Folder...**.
- В строке пути вверху введите путь к папке:
  - Если будете клонировать из GitHub (Способ А) — пока откройте домашнюю папку: **`/home/ubuntu`** (или `/home/ваш_логин`, если логин другой). Нажмите **Enter**.
  - Если будете загружать файлы вручную (Способ Б) — тоже откройте **`/home/ubuntu`** (или вашу домашнюю папку). Нажмите **Enter**.

## ЧАСТЬ 3. Появление папки с ботом на сервере

### Если выбрали Способ А (GitHub)

1. В Cursor в окне с сервером откройте терминал: меню **Terminal** → **New Terminal** (или **Ctrl+`**).
2. В терминале внизу вы должны быть в какой-то папке (например `/home/ubuntu`).
3. Введите команду (замените ссылку на **ваш** репозиторий):
   ```bash
   git clone https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПО.git davai_sigrai_bot
   ```
   Пример: `git clone https://github.com/ivan/davai_sigrai_bot.git davai_sigrai_bot`
4. Если попросит логин и пароль GitHub — введите их (пароль при вводе не виден — это нормально).
5. Когда команда выполнится, появится папка **davai_sigrai_bot**. Путь к боту будет: **`/home/ubuntu/davai_sigrai_bot`** (или `/home/ваш_логин/davai_sigrai_bot`).
6. В Cursor: **File → Open Folder** и укажите **`/home/ubuntu/davai_sigrai_bot`** (или ваш путь). Нажмите OK.

### Если выбрали Способ Б (без GitHub)

1. В окне с сервером в левой панели (проводник) найдите открытую папку (например `ubuntu`).
2. Правой кнопкой по пустому месту → **New Folder** → имя папки: **davai_sigrai_bot** → Enter.
3. Откройте **второе окно** Cursor: **File → New Window**.
4. Во втором окне откройте **вашу локальную** папку с ботом (с компьютера): **File → Open Folder** → выберите папку `davai_sigrai_bot` с рабочего стола (или где она у вас).
5. В этом втором окне в левой панели выделите **все** файлы и папки (Ctrl+A): main.py, bot, config, data, run_bot.sh, deploy_on_server.sh, .env.example и т.д.
6. Перетащите их мышкой в первое окно (сервер) в папку **davai_sigrai_bot**.
7. В первом окне (сервер) закройте текущую папку и откройте папку с ботом: **File → Open Folder** → введите **`/home/ubuntu/davai_sigrai_bot`** (или ваш путь) → OK.

## ЧАСТЬ 4. Запуск скрипта установки на сервере

Сейчас у вас в Cursor открыта **папка с ботом на сервере** (в ней видны main.py, bot, config, data, run_bot.sh и т.д.).

### Шаг 4.1. Открыть терминал на сервере

- Меню **Terminal** → **New Terminal** (или нажмите **Ctrl+`**).
- Внизу откроется терминал. В нём должен быть путь к папке бота (например `/home/ubuntu/davai_sigrai_bot`). Если нет — введите:
  ```bash
  cd /home/ubuntu/davai_sigrai_bot
  ```
  (или ваш путь к папке с ботом) и нажмите Enter.

### Шаг 4.2. Узнать своё имя пользователя на сервере

- В том же терминале введите:
  ```bash
  whoami
  ```
- Нажмите Enter. Появится одно слово — например **ubuntu** или **root**. Это и есть имя пользователя — запомните его.

### Шаг 4.3. Запустить скрипт установки

- Введите (подставьте **своё** имя пользователя вместо `ubuntu`, если у вас другое):
  ```bash
  chmod +x deploy_on_server.sh
  ```
  Нажмите Enter.

- Затем введите:
  ```bash
  ./deploy_on_server.sh ubuntu
  ```
  (вместо `ubuntu` подставьте то, что показал `whoami`). Нажмите Enter.

- Скрипт будет что-то писать и может попросить **пароль** (для sudo). Введите пароль от сервера и нажмите Enter (символы не видны — нормально).
- Дождитесь сообщения вроде «Готово» или окончания работы скрипта.

Если появятся красные строки с ошибкой — **скопируйте их целиком** и сохраните: по ним можно будет понять, что не так.

## ЧАСТЬ 5. Вписать токен бота

### Шаг 5.1. Открыть файл .env

- В левой панели Cursor (проводник) найдите файл **.env** (он может быть в корне папки бота).
- Если файла **.env** нет — найдите **.env.example**, откройте его, затем **File → Save As** и сохраните как **.env** в той же папке.
- Откройте файл **.env** двойным щелчком.

### Шаг 5.2. Вставить свой токен

- В файле будет строка вида:
  ```
  BOT_TOKEN=your_bot_token_here
  ```
- Замените **your_bot_token_here** на **ваш настоящий токен** от @BotFather (длинная строка вроде `8519012332:AAH132h9QFN6n1ymxHX3oea3N2A-0y9lq6U` — это пример, вставьте свой).
- Оставьте формат как есть: `BOT_TOKEN=ваш_токен`, без пробелов вокруг `=`.
- Сохраните файл: **Ctrl+S**.

### Шаг 5.3. Перезапустить бота на сервере

- Снова откройте терминал внизу (или используйте тот же).
- Введите и нажмите Enter:
  ```bash
  sudo systemctl restart davaj_igrat_bot.service
  ```
- Если попросит пароль — введите пароль от сервера и нажмите Enter.

---

## ЧАСТЬ 6. Проверить, что бот работает

- В том же терминале введите:
  ```bash
  sudo systemctl status davaj_igrat_bot.service
  ```
- Нажмите Enter.

- Хороший вариант: в тексте есть строка **active (running)** зелёным. Значит, бот запущен с сервера.
- Плохой вариант: **failed** или **inactive**. Тогда введите:
  ```bash
  tail -30 /home/ubuntu/davai_sigrai_bot/logs/bot_stderr.log
  ```
  (если папка бота у вас в другом месте — замените путь). Скопируйте вывод и по нему можно будет искать причину.
